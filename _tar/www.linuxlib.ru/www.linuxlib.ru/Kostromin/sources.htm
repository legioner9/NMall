П1.1. Книги по UNIX и Linux

Ниже приводится небольшой список книг, которые на мой взгляд стоит прочесть. Приводятся не только книги по Linux, но и книги, посвященные UNIX вообще. Дело в том, что Linux - это один из вариантов UNIX-а, поэтому из этих книг можно почерпнуть очень много полезного. Кроме того, в настоящей книге, как и в любой другой, невозможно привести весь необходимый материал, особенно по внутренней структуре и организации ОС, поэтому если Вы хотите глубоко разобраться в устройстве Linux, прочитать эти книги просто необходимо.
Список не претендует на библиографическую полноту, приводятся только книги, с которыми я имел возможность познакомиться и которые полезны на мой взгляд (по моему скромному мнению).

Книги по UNIX

1. Стен Келли-Бутл "Введение в UNIX", перевод с английского С.Орлова, изд. "ЛОРИ", 1995 г.
Очень полезная книга для начинающих пользователей. Здесь Вы найдете материал по тем вопросам, которые не нашли освещения в моей работе: описание основных команд UNIX, файловой системы, классических текстовых редакторов ed и vi и других, сценариев командного процессора, основ системы X Window, основ системного администрирования и т.д.

2. М.Бах "Архитектура операционной системы UNIX" (THE DESIGN OF THE UNIX OPERATING SYSTEM by Maurice J. Bach), перевод с английского к.т.н. Крюкова А.В.
Прочитав эту книгу, Вы поймете внутренние механизмы UNIX (и Linux). Широко распространена в Интернет в электронных версиях.

3. Э.Немет и др. "UNIX. Руководство системного администратора" Перевод с английского С.М.Тимачева, 4-ое изд. "BHV", Киев, 1999 г. Лучшая из книг, посвященных системному администрированию Unix. По Linux там, конечно, ничего нет, но прочитать эту книгу стоит (это не только мое мнение, я слышал такие отзывы от нескольких системных администраторов UNIX).

4. Робачевский А.М., Операционная система Unix. - СПб.: BHV- Санкт-Петербург, 1998.-528 с., ил. Книга часто упоминается в ссылках, так что, наверное, ее стоит прочитать (но я ее не видел). В RU.LINUX.FAQ пишут: "ОС UNIX" Андрея Робачевского - это почти общепризнанно сборник плохих переводов хороших оригиналов. К сожалению, альтернативы этой книге нет, пока не будут переведены и/или (пере)изданы сами оригиналы, перечисленные в конце книжки: Бах, Стивенс, Лефлер."

Книги по Linux

1. Ричард Петерсен "LINUX: руководство по операционной системе", перевод с английского С.М.Тимачева, изд. BHV, Киев, 2-ое изд., 1999 г. (в комплекте с CD-ROM).
Эта книга описывает дистрибутив Caldera, но я очень активно использовал эту книгу при изучении Red Hat Linux. Она охватывает все основные вопросы работы в системе, и хотя изложение несколько сумбурное, но пролистать ее полезно.

2. Д.Такет (мл.), С.Барнет "Использование Linux. Полное справочное руководство. Специальное издание", пер. с англ. 4-е издание, Издательский дом "Вильямс", Киев, Москва, Санкт-Петербург. 1999 г.
Очень толстая книга, частично от того, что один и тот же материал повторяется дважды (например, дан материал по установке Linux вообще, а потом то же самое почти дословно повторяется для RedHat). Но лучшей книги по Linux я пока в руках не держал.

3. Red Hat 6.2 Linux. Учебный курс. Под редакцией А.Пасечника. Изд. "Питер", Санкт-Петербург, 2000 г.
Эта книга представляет собой перевод двух руководств: "Официального руководства по установке и конфигурированию Red Hat Linux 6.2" и "Официального руководства начинающего пользователя Red Hat Linux". Если первая из этих двух частей очень подробна и детальна, то вторая по необходимости очень конспективна. Единственное (кроме процедур установки, естественно), что в этой книге рассмотрено подробнее, чем в других книгах по Линукс (из тех, что я читал), - это работа с программой linuxconf. Ну, и еще Приложение D - перечень входящих в дистрибутив пакетов (на 133 страницах!), который полезен как справочное пособие.

4. Владимир Водолазский "Путь к Linux". Эта книга, говорят, хороша для начинающих (печатное издание 99-го года, а не электронное 97-го)

    Разработки отдельных пакетов ПО:

    kernel.org
    GNOME

    Beowulf is a project with an aim of making software to run off-the-shelf clustered workstations based on standard PC hardware, a high speed network, and Linux.
    DOSEMU is a DOS Emulator for Linux. It can run most applications seamlessly on a Linux environment, either remotely or locally in a terminal or an X-Term.
    GNU (GNU's Not UNIX) has released many common applications for Linux, such as the ever famous Emacs. However, their goal is to develop another free UNIX, similar to Linux, however bearing the name GNU.
    KDE (K Desktop Environment) is an object oriented desktop environment for X-Windows. It seems very advanced, with desktop web browsers and the likes built into the system.
    The Linux Advocacy Project is attempting to make developers consider porting their programs to Linux and other UNIX environments. They are also widely advocating the use of Linux in the general community.
    The Mnemonic Browser when it is completed will be an X-based GPL'ed web browser, that will hopefully be commercial standard. It's looking very promising.
    (10.09.99) WINE people are working towards a Windows emulator for Linux/X. So far it can run Solitare, Winword and lots of applications, although more work needs to be done. Eventually it will support 32-bit Windows applications as well.
