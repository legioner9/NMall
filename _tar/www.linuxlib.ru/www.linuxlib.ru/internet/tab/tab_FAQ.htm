  Данный документ представляет собой список часто задаваемых вопросов
  по биллинговой системе TA Billing от проекта Traffic Accounting    
  

  [Вопрос]:

    Возможно ли использование комплекса TA Billing для
    подсчета трафика коммерческих клиентов(провайдинг) ?
    [Ответ]:
      Да, возможно.


  [Вопрос]:

    Будет ли работать TA Billing под FreeBSD ?
    [Ответ]:
      Нет, TA Billing предназначен только для Linux.


  [Вопрос]:

    Планируется ли портировать TA Billing под FreeBSD ?
    [Ответ]:
      Нет, портирования не планируется.


  [Вопрос]:

    После ввода имени, пароля, и нажатия кнопки "Войти"
    выводится та же страница входа.
    [Ответ]:
      Исправьте php.ini:
      register_globals = On
      short_open_tag = On


  [Вопрос]:

    После исправления php.ini никакого результата
    не происходит.
    [Ответ]:
      С помощью функции phpinfo(), узнайте
      откуда php берет файл php.ini,
      далее именно этот файл и надо изменять.


  [Вопрос]:

    Как запустить TA Billing ?
    [Ответ]:
      В подкаталоге bin, командой rta.


  [Вопрос]:

    Как можно остановить/перезапустить TA Billing ?
    [Ответ]:
      командой rta в подкаталоге bin.
      rta start, для запуска, тоже самое если rta без параметров.
      rta stop, для остановки.
      rta restart, для перезапуска.


  [Вопрос]:

    При добавлении пользователя в TA Web,
    выдает ошибку - "нет прав доступа".
    [Ответ]:
      Допустим для TA Web используется
      каталог /home/ta.
      Установите на все файлы и директории,
      пользователя равного пользователю под которым
      работает apache, например:
      chown apache.apache -R /home/ta

      Или же если вы используете хостинг,
      и к папке /home/ta могут получить
      несанкционированный доступ то
      в таком случае вам необходимо
      поставить патч к apache,
      патч называется php-suexec,
      ссылку на этот патч можно
      найти на поисковиках.
      После установки патча вы просто
      устанавливаете права на все файлы
      в папке такими какими они должны быть:
      chown ta.ta -R /home/ta


  [Вопрос]:

    При первом заходе в TAB выводится ошибка
    "Немогу создать файл с индексами".
    [Ответ]:
      см. предыдущий вопрос.


  [Вопрос]:

    Под какими ядрами работает система ?
    [Ответ]:
      Под ядрами не ниже 2.2.X


  [Вопрос]:

    С каким файрволом работает TA Billing,
    iptables или ipchains ?
    [Ответ]:
      TA Billing не привязан ни к одному файрволу,
      вы указываете путь у файрволу только для
      инсталляции, после инсталляции создается
      скрипт kshell, для отключения/подключения
      пользователей, в нем прописаны указания
      изначально для iptables, потом вы можете
      изменить этот скрипт под свои нужды, более
      iptables ни для чего не указывается кроме
      скрипта kshell.
      Таким образом, TA Billing вообще никак не
      использует ни iptables, ни ipchains,
      а iptables указывается только для начала
      инсталляции, для примера в скрипте kshell.


  [Вопрос]:

    Возможна ли раздельная тарификация россия/зарубеж ?
    [Ответ]:
      Да возможна, для этого существует
      раздел "Разделение сетей" в TA Web.


  [Вопрос]:

    Возможно ли в одной конфигурации установить различные
    тарифы на использование локального и внешнего трафика ?
    [Ответ]:
      Да, возможно.
      Для этого нужно будет воспользоваться
      разделом "Разделение сетей".
      Ваша конфигурация должна будет
      обрабатывать трафик только
      локального интерфейса(alo_dev=(ЛОКАЛЬНЫЙ_ИНТЕРФЕЙС)).
      Далее для того чтобы определить тариф
      на трафик локальной сети необходимо
      в разделе "Разделение сетей"
      добавить сеть, например b192.168.0.0/24,
      добавлять нужно обязательно с префиксом "b".
      Далее если необходимо разделение трафика
      на российский и зарубежный то добавлять
      сети с префиксом "b" не нужно.


  [Вопрос]:

    Возможно ли автономное использование программы TA из TA Billing ?
    [Ответ]:
      Да, возможно.
      Для этого нужно только использовать
      ta.conf как после инсталляции TA Billing.
      А при использовании конфигураций
      необходимо будет использовать функции
      update_*. При этом если не предполагается
      использование TA Web, то функции taw_*
      можно исключить из конфигурационного файла.


  [Вопрос]:

    Скрипт install выдает ошибку - "./install :${NAME:0:1} bad substitution".
    [Ответ]:
      В этом случае необходимо обновить bash.
      Так же возможно попробовать заменить
      первую строку файла ./install, "#!/bin/bash"
      на "#!/bin/bash2".
