LPADMIN(8)

НАЗВАНИЕ
lpadmin - устанавливает конфигурацию системы спулинга печати.

СИНТАКСИС 


/usr/lib/lpadmin -p printer [options...]

/usr/lib/lpadmin -x dest

/usr/lib/lpadmin -d[dest]

ОПИСАНИЕ
lpadmin конфигурирует систему спулинга печати, описывая принтеры, классы и устройства. Она используется для добавления и удаления назначений, изменения членского состава классов, изменения устройств, связанных с принтерами, и интерфейсных программ печати, а также для изменения назначения, устанавливаемого в системе по умолчанию. Администратор системы может использовать lpadmin также для добавления к системе новых назначений печати. lpadmin нельзя использовать при работающем планировщике печати lpsched(8), за исключением указанных ниже случаев.

Для правильного запуска lpadmin нужно использовать только одну из опций -p, -d или -x.

-d[dest]
Задает новое системное назначение dest, используемое в системе по умолчанию. Если dest не задано, тогда системное назначение, используемое по умолчанию, отсутствует. Эта опция может использоваться при работающем lpsched(8). Совместное использование с другими опциями не разрешается.
-xdest
Удаляет назначение dest из системы печати. Если dest принтер, единственный в классе, тогда удаляется и этот класс. Совместное использование с другими опциями не разрешается.
-pprinter
Называет имя printer, к которому относятся приведенные ниже опции. Если printer не существует, то он будет создан. Совместное использование с другими опциями не разрешается.

Следующие опции используется только совместно с опцией -p и могут набираться в любом порядке. Для простоты принтер будет называться ниже p.

-cclass
Вводит принтер p в заданный класс. Если класс class не существует, то он будет создан.
-eprinter
Копирует существующую интерфейсную программу, соответствующую устройству printer, в качестве новой интерфейсной программы для p.

Указывает, что устройство, связанное с p, присоединено. Задание этой опции подразумевается при подключении нового принтера, если не задана опция -l.

Устанавливает новую интерфейсную программу для p. Interface - полное имя этой программы.

Указывает, что устройство, связанное с p, является подключаемым терминалом. Планировщик печати lpsched(ADM), стартуя, отключает все терминалы автоматически. Перед тем, как снова активизировать p, нужно установить его текущее устройство device с помощью lpadmin.

Версия интерфейсной программы печати dumb поставляется вместе с программными средствами печати системы XENIX. Она представляет собой процедуру-оболочку, которая служит интерфейсом между lpsched и устройствами печати. Ее можно найти в каталоге /usr/spool/lp/model и использовать в поставляемом виде с помощью lpadmin -p или lpinit(8). Эта программа служит интерфейсом к принтеру без специальных функций и протоколов. Предполагается обработка символов перевода на новую страницу. Администратор системы может модифицировать копию dumb и затем связать ее с принтером с помощью lpadmin -i.

Удаляет принтер p из заданного класса class. Если p последний член класса, то удаляется этот класс.

Связывает новое устройство device с принтером p. Device - это полное имя файла, который доступен для записи программе lp. Заметьте, что ничто не мешает lp связать одно и то же устройство device с несколькими принтерами. Если задаются только опции -p и -v, то lpadmin можно использовать при работающем планировщике.

ОГРАНИЧЕНИЯ
При подключении нового принтера должны задаваться опции -v и одна из -e, -i или -m. Из последних трех опций можно задать только одну. Ключевые опции -h и -l являются взаимоисключющими. Имена принтеров и классов должны быть не длиннее строки из 14 символов и могут состоят только из символов A - Z, a - z, 0 - 9 и _(символ подчеркивания).

ИНТЕРФЕЙСНЫЕ ПРОГРАММЫ ПЕЧАТИ
Эти программы являются процедурами-оболочками, которые служат интерфейсами между lpsched(8) и устройствами. Они располагаются в каталоге /usr/spool/lp/model и могут быть использованы с помощью lpadmin -m. Если ими владеют lp & bin, то они должны иметь 644 разрешения, а если ими владеют bin & bin, то - 664 разрешения. Системный администратор может модифицировать их копии и затем связывать их с принтерами с помощью lpadmin -i. Если принтеры имеют специальные опции, то их можно включать в интерфейсную программу.

После этого пользователи могут выбирать опции с помощью команды lp -o. Одна версия интерфейсной программы поставляется вместе с программными средствами печати системы UNIX. Это - программа dumb, которая представляет из себя интерфейс к принтеру без специальных функций и протокола. Предполагается обработка символов перевода на новую строку. Это хорошая программа, которую системный администратор может скопировать и модифицировать.
