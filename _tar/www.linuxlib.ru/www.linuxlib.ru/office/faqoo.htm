OpenOffice.ru FAQ

(версия 0.0.4)

В этом документе сделана попытка собрать ответы на наиболее типичные вопросы новых пользователей пакета OpenOffice.org, которые обсуждались в конференции [email protected] .

Я нисколько не претендую на авторство ответов, а выступаю только в роли собирателя.

Так как OpenOffice.org работает на различных платформах, в начале вопроса, специфического для конкретной OS она указана явным образом.

Установка

В: Linux. Начинаю устанавливать новую версию -- появляется надпись Linux glibc 2.2.4, бежит полосочка распаковки архива и после этого все умирает.
О: Проверьте наличие свободного места на диске.

В: Linux. Перед установкой новой версии OpenOffice.org старая была удалена из каталога /opt/OpenOffice.xxx. Однако с установкой возникают проблемы.
О: Удалите в домашнем каталоге .sversonrc и .user60.rdb.

В: При установке появляется окно "среда выполнения Java". А у меня Java нет, OpenOffice.org не будет работать?
О: Java нужна для использования Java-апплетов в документах и установке некоторых plugins ОО. Если вы не планируете использование Java-апплетов, то она вам не потребуется. ОО прекрасно работает без нее. При необходимости среду выполнения Java можно поставить позже.

В: Linux. В процессе установки вместо русских букв на экране в диалогах знаки вопроса.
О: Для нормальной работы OpenOffice.org необходимы русифицированные шрифты (Type1 или TrueType) в кодировке microsoft-cp1251. Убедитесь, что подобные шрифты есть в Вашей системе до начала установки.
На момент написания FAQ шрифты MS Typography недоступны с сайта MS, однако их можно скачать с http://corefonts.sourceforge.net , а также использовать шрифты Валентина Филиппова с ftp://ftp.gnome.ru.

В: Linux. После установки возникают проблемы с правильным отображением русских шрифтов и печатью.
О: OpenOffice.org необходимо явно указать имеющиеся у Вас шрифты. Запустите программу ./spadmin Нажмите кнопку <Шрифты>, потом кнопку <Добавить> и укажите путь к каталогу со шрифтами Type1 или TrueType. Нажмите кнопку <Выбрать все> и подключите шрифты к OpenOffice.org кнопкой <OK>. Учтите, что при наличии в этом каталоге дефектных шрифтов, может произойти аварийное завершение работы программы spadmin. В этом случае удалите из подключаемого каталога некачественные шрифты.

Шрифты

В: Мне не нравится шрифт в заголовках меню и диалоговых окнах. Как я могу его поменять?
О: По умолчанию для отображения текста в меню и диалоговых окнах используется шрифт Andale Sans UI.
Необходимо заменить шрифт, для этого откройте диалоговое окно Сервис -> Параметры -> OpenOffice.org -> Замена шрифта, включите <Применить таблицу замен>, в поле <Тип шрифта> вручную наберите Andale Sans UI, в поле <Заменить на> выберите нужный шрифт.

В: Linux:У меня в меню и диалоговых окнах вместо русского текста пустые места (или непонятные символы).
О: Установите шрифты microsoft-cp1251 (см. в разделе Установка).

В: Linux: Я брал шрифты Microsoft из Windows NT с NTFS раздела, но эти шрифты не видны, на месте текста пустые места.
О: У этих шрифтов права root.root 600. Поменяйте права на шрифты.

Импорт документов

В: Я не могу открыть в OpenOffice.org простой текстовый файл. Как это сделать?
О: При открытии подобных файлов выберите тип файла - Текст кодированный. OpenOffice.org запросит необходимую кодировку - укажите ее.

В: При открытии документов MS Word 95 и MS Excel 95 русский текст неправильно отображается.
О: Это известная проблема MS Office. Для перекодировки используйте макросы Сервис -> Макрос ->Макрос из -> Tools <RecodeCyrDoc> для текста и <RecodeCyrSheet> для таблиц.

Работа с OOwriter

В: Как расставить номера страниц?
О: Вставьте нижний (или верхний) колонтитул -> Вставка -> колонтитул, а затем Вставка-> Поле-> Номер страницы.

В: Как избавиться от номера на первой странице?
О: Ей надо задать стиль первой страницы. Откройте стилиста <F11>, в ряду пиктограмм выберите стиль страницы и задайте нужный стиль.

В: В заголовке документа отражаются непонятные символы или первая строка документа.
О: Вы можете отредактировать заголовок документа по своему усмотрению. Файл -> Свойства - > Описание -> Заголовок.

В: Как сделать, чтобы в заголовке отображалось имя файла?
О: Необходимо выбрать в меню Файл -> Перезагрузить. Внимание! Это нужно делать перед редактированием файла или после его сохранения.

В: Скажите, а есть ли в OpenOffice.org возможность проверки русской орфографии?
О: Да, такая возможность есть в российских сборках OpenOffice.org, например от ALTLinux. По умолчанию модули проверки орфографии и расстановки переносов выключены.
Активизировать модули проверки русской орфографии и расстановки переносов для русского языка вы можете, выбрав "Сервис -> Параметры -> Настройки языка -> Лингвистика". Нажмите кнопку "Правка" и поставьте галочки напротив нужных модулей.
Если у вас была установлена предыдущая версия OpenOffice.org, обязательно удалите ее полностью, иначе могут быть проблемы с активизацией модуля орфографии.

В: Вопрос - существует ли в OOo защита документов паролем ?
О: Да, существует. Файл -> Сохранить Как ..., отметить "Сохранить с паролем". При этом пароль следует задавать латинскими буквами или цифрами, кириллица пока не работает.

В: Можно ли в OpenOffice.org для каждой страницы документа задать свой стиль офомления.
О: Это возможно. Включите стилиста, если он выключен (F11) и далее составьте необходимое количество нужных стилей страниц (например, один альбомный, другой -- книжный и т.д.). Стили страниц доступны при нажатии на кнопку в верхней части стилиста с изображением листа с завернутым углом. Затем просто назначайте нужный стиль на нужную страницу.

В: Меня не устраивает шаблон, загружаемый по умолчанию. Как я могу его изменить
О: Для этого сохраните необходимый файл, как шаблон "Файл -> Шаблон документа ->Сохранить..." и организуйте его как стандартный шаблон. "Файл -> Шаблон документа -> Организовать...", затем выбрать в левом окне сохраненный шаблон и нажать на кнопку "Команды", затем выбрать пункт "Как стандартный шаблон".


Базы данных

В: В OpenOffice.org не входит СУБД. Как я могу работать с данными?
О: Действительно, СУБД не входит в OpenOffice.org, однако достаточно развитые средства работы с данными имеются. Можно работать с локальными данными, представленными в виде текстовых и табличных файлов и файлов dbase, а также с базами данных через odbc и jdbс. Смотрите Сервис -> Исходные данные.

В: При первом запуске OpenOffice.org создал адресную книгу. Как я могу ее использовать?
О: Проверьте, есть ли она в <Исходных данных>. Если надо - сделайте запросы. Потом: Вставка -> Поле -> Дополнительно -> База данных. Выберите базу данных, запрос (при необходимости), вставьте <поле циркулярного письма> и отметьте нужные поля базы. Сохраните документ. А теперь нажмите Файл - > Циркулярное письмо. Отметьте нужные записи. ОО начнет печатать рассылку по выбранным адресам.

В: Linux. Где можно получить дополнительную информацию по jdbc?
О: Посмотрите этот документ http://www.unixodbc.org/doc/OOoMySQL.pdf
